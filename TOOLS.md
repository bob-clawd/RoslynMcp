# Tool descriptions

| Tool | Description | Parameters | Outputs |
| ---- | ----------- | ---------- | ------- |
| `load_solution` | Use this tool when you need to start working with a .NET solution and no solution has been loaded yet. This must be the first tool called in a session before any code analysis or navigation tools can be used. | `solutionHintPath` (optional): Absolute path to a `.sln` file, or to a directory used as the recursive discovery root for `.sln`/`.slnx` files. If omitted, the tool auto-detects from the current workspace. | Loaded solution `path` plus `projects` and `edges` summaries. `projects` is bucketed into `libraries`, `executables`, and optional `unknown`, each with `leaves` / `intermediates` / `roots`. `projects.count` is the number of emitted file-backed projects. `edges` contains `count`, `cycleDetected`, and the explicit `references` edge list. |
| `load_project` | Use this tool when you need to list types declared in a specific project. It is useful for project-scoped discovery, for finding type symbols before follow-up calls such as `load_type` or `load_member`. | `projectPath`: Project path from `load_solution` (relative or absolute `.csproj` path), or a loaded project name such as `ProjectImpl`. | Type count plus project-local types. Each entry contains a `type` symbol and its declared `members` count. |
| `load_type` | Use this tool when you need to inspect type hierarchy and members declared by the specific type. | `typeSymbolId`: The stable symbol ID of a type, obtained from `load_project` or from a `search_type` match. | The resolved type symbol, optional documentation summary, base types, interfaces, derived types, implementations, and declared handwritten members. |
| `load_member` | Use this tool when you need callers/callees or overrides/implementations of a symbol. | `memberSymbolId`: The stable symbol ID, obtained from `load_type` or from a `search_member` match. | The resolved member symbol, documentation, direct callers, direct callees, overrides, and implementations. |
| `find_references` | Use this tool when you need to see which files and containing types reference a symbol. | `symbolId`: The stable symbol ID of a type or member. | Complete reference contexts with `filePath` and optional `containingTypeSymbolId`. Each entry represents one affected `(file, containing type)` context, not every individual occurrence. |
| `search_member` | Use this tool when you only have a member name fragment (method/property/event/ctor). It performs a fast syntax-only search over the loaded solution. If exactly one match is found, it automatically returns the `load_member` result for that symbol. | `query`: Member name fragment to search for (case-insensitive contains match). | If ambiguous: returns up to 50 matches with `fullName`, `projectPath`, `location`, `kind`, optional `signature`, and `memberSymbolId` for up to the first 10 matches. When omitted, the id was not semantically resolved. When the list is cut off, the response also includes `truncated: true`. If unique: `member` contains an embedded `load_member` result. |
| `run_tests` | Default .NET test runner. Use this instead of `dotnet test` unless you need unsupported CLI behavior. | `target` (optional): Execution target. Omit to run the currently loaded solution, or the workspace directory when no solution is loaded. Supports workspace-relative or absolute `.sln`, `.slnx`, `.csproj`, or directory paths when the resolved target stays within the workspace directory.<br><br>`filter` (optional): `dotnet test` filter expression, passed through where practical. | Test run outcome, exit code, summary, grouped test failures, optional build diagnostics, and aggregated test counts. |
| `search_type` | Fast, syntax-only type discovery across the loaded solution. Use this when you only have a type name fragment and want to avoid grep/glob. If exactly one match is found, it automatically returns the `load_type` result for that type. | `query`: Type name fragment (case-insensitive contains match). | If exactly one match: `type` contains a `load_type` result. If multiple matches: returns up to 50 `matches` with `fullName`, `projectPath`, and `typeSymbolId` for up to the first 10 matches. When omitted, the id was not semantically resolved. When the list is cut off, the response also includes `truncated: true`. |
| `diagnose_file` | Get compiler error diagnostics for one source file without reloading the solution or running tests. | `filePath`: Path to a source file within the currently loaded solution. Accepts an absolute path or a path relative to the workspace. | Up to 10 file-local compiler errors with `line`, `column`, `id`, and `message`. |
